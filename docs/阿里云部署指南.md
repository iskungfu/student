# 学生管理系统 - 阿里云部署指南

## 一、项目概述

学生管理系统是一个基于 Vue3 + Express + MySQL 的前后端分离项目，包含学生管理、班级管理、专业管理、宿舍管理、菜单管理和系统管理等模块。

**技术栈：**
- 前端：Vue3 + Element Plus + Vue Router + Pinia + Axios
- 后端：Express + MySQL2 + JWT
- 数据库：MySQL 8.0（阿里云）

## 五、前端部署

### 5.1 上传前端代码

将 `client` 目录上传到服务器，例如 `/var/www/student-management-client`

```bash
# 创建项目目录
sudo mkdir -p /var/www/student-management-client

# 上传文件
scp -r client/* user@47.86.55.121:/var/www/student-management-client/
```

### 5.2 安装依赖

```bash
cd /var/www/student-management-client
npm install
```

### 5.3 修改 API 地址

编辑 `vite.config.js`，修改 API 代理配置：

```javascript
export default defineConfig({
  plugins: [vue()],
  server: {
    port: 5173,
    proxy: {
      '/api': {
        target: 'http://47.86.55.121:3000',  // 修改为服务器地址
        changeOrigin: true
      }
    }
  }
})
```

### 5.4 构建生产版本

```bash
npm run build
```

构建完成后，会生成 `dist` 目录。

### 5.5 配置 Nginx

编辑 Nginx 配置文件：

```bash
sudo vi /etc/nginx/conf.d/student-management.conf
```

添加以下配置：

```nginx
server {
    listen 80;
    server_name 47.86.55.121;  # 或你的域名

    # 前端静态资源
    location / {
        root /var/www/student-management-client/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 5.6 重启 Nginx

```bash
# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

## 六、防火墙配置

### 6.1 开放端口

```bash
# 开放 80 端口（HTTP）
sudo firewall-cmd --permanent --add-port=80/tcp

# 开放 3000 端口（后端 API）
sudo firewall-cmd --permanent --add-port=3000/tcp

# 重载防火墙
sudo firewall-cmd --reload
```

### 6.2 或者关闭防火墙（测试环境）

```bash
sudo systemctl stop firewalld
sudo systemctl disable firewalld
```


```nginx
server {
    listen 443 ssl http2;
    server_name yourdomain.com;

    ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;

    # ... 其他配置
}

server {
    listen 80;
    server_name yourdomain.com;
    return 301 https://$server_name$request_uri;
}
```

## 九、常见问题

### 9.1 PM2 常用命令

```bash
# 列出所有进程
pm2 list

# 查看详细信息
pm2 show student-api

# 查看日志
pm2 logs student-api

# 重启服务
pm2 restart student-api

# 停止服务
pm2 stop student-api

# 删除进程
pm2 delete student-api
```

### 9.2 数据库连接失败

检查防火墙是否允许数据库端口（默认 3306）：

```bash
sudo firewall-cmd --permanent --add-port=3306/tcp
sudo firewall-cmd --reload
```

### 9.3 Nginx 502 错误

检查后端服务是否运行：

```bash
pm2 status
pm2 logs student-api
```

### 9.4 查看系统日志

```bash
# Nginx 日志
sudo tail -f /var/log/nginx/error.log

# PM2 日志
pm2 logs --lines 100
```
